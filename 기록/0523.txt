- component-scan 이란?

어노테이션을 클래스에 선언하면 스프링이 Bean으로 등록 될 준비를 마친 클래스들을 스캔하여

디폴트로 @Component @Controller @Service @Repository 를 Bean으로 등록해준다.

 

- component-scan 사용법

<context:component-scan base-package="패키지명"/>
위와 같이 설정하고 base package를 적으면 base package를 기준으로 클래스를 스캔해 빈으로 생성한다.		
		
- component-scan 동작 과정

ConfigurationClassParser 가 Configuration 클래스를 파싱한다.
@Configuration 어노테이션 클래스를 파싱하는 것이다.
                   
ComponentScan 설정을 파싱한다.
base-package 에 설정한 패키지를 기준으로
ComponentScanAnnotationParser가 스캔하기 위한 설정을 파싱한다.
                   
base-package 설정을 바탕으로 모든 클래스를 로딩한다.
                  
ClassLoader가 로딩한 클래스들을 BeanDefinition으로 정의한다.
생성할 빈의 대한 정의를 하는 것이다.
                   
생성할 빈에 대한 정의를 토대로 빈을 생성한다.

* mybatis-spring:scan 역할 ??
일반적으로 MyBatis에서 매퍼 인터페이스는 데이터베이스 테이블 또는 엔티티에 대한 CRUD(Create, Read, Update, Delete) 작업을 수행하기 위해 사용됩니다. 매퍼 인터페이스는 SQL 문장을 실행하는 메서드를 정의하고, 각 메서드는 데이터 조작에 필요한 파라미터를 받아 해당 동작을 수행합니다.

@Mapper 어노테이션은 MyBatis와 스프링의 연동을 위해 사용되는 어노테이션입니다. 이 어노테이션은 MyBatis가 해당 인터페이스를 매퍼로 인식하고, 자동으로 매퍼 인스턴스를 생성하여 스프링 컨테이너에 등록해줍니다. 따라서 UserMapper 인터페이스에 @Mapper 어노테이션이 지정되어 있으므로, 이 인터페이스를 사용하기 위해 별도의 설정 없이 스프링에서 매퍼 인스턴스를 주입받을 수 있습니다.
매퍼 인터페이스는 MyBatis에서 제공하는 XML 기반의 매퍼 파일과 대체될 수 있습니다. XML 매퍼 파일에는 매퍼 인터페이스와 동일한 SQL 문장들이 정의되어 있으며, 해당 인터페이스와 매퍼 파일은 일대일로 매핑될 수 있습니다. 매퍼 인터페이스를 사용하면 SQL 문장들을 자바 코드로 직접 작성하고 호출할 수 있으며, 매퍼 파일을 사용하면 SQL 문장들을 별도의 XML 파일에 작성하고 매퍼 인터페이스와 연결할 수 있습니다. 어느 방식을 선택할지는 개발자의 선호나 프로젝트 요구사항에 따라 결정됩니다.

1. SqlSession을 주입받을 빈이 @Autowired 어노테이션 또는 생성자 주입을 통해 정확하게 주입되는지 확인합니다. 위의 예제에서는 BoardDAOMyBatis 클래스의 생성자에서 SqlSession을 주입받고 있습니다. 이때 SqlSession을 주입받기 위한 빈이 스프링 컨테이너에 등록되어 있어야 합니다.

2. SqlSession을 주입받을 빈을 스프링 설정 파일이나 @Configuration 어노테이션이 적용된 클래스에서 등록해야 합니다. 예를 들어, SqlSession을 생성하는 팩토리 빈을 설정 파일에 등록하거나, @MapperScan 어노테이션이 적용된 클래스에서 SqlSession을 주입받을 수 있도록 설정해야 합니다.

3. MyBatis의 설정과 관련하여 주의해야 할 점은 SqlSession을 생성하기 위한 SqlSessionFactory 빈이 스프링 컨테이너에 등록되어 있어야 한다는 것입니다. SqlSessionFactory는 SqlSession을 생성하는 팩토리 역할을 담당합니다. 이를 위해 MyBatis의 설정 파일인 mybatis-config.xml을 작성하고, 해당 설정 파일을 사용하여 SqlSessionFactoryBean 빈을 등록해야 합니다.



@Configuration 어노테이션은 스프링에서 설정 클래스를 나타내는 어노테이션입니다. 이 어노테이션을 사용하여 클래스를 설정 클래스로 지정하면, 해당 클래스는 스프링의 빈 구성 요소로 사용됩니다.

@Configuration 어노테이션을 클래스에 적용하면 다음과 같은 효과가 있습니다:

1. 스프링 컨테이너에 의해 빈으로 인식됩니다: @Configuration 어노테이션이 적용된 클래스는 스프링 컨테이너에 의해 빈으로 인식되어 관리됩니다. 다른 빈과 마찬가지로 컨테이너가 생성하고 초기화하여 애플리케이션에서 사용할 수 있습니다.

2. 빈의 정의를 포함하는 메서드를 정의할 수 있습니다: @Configuration 어노테이션이 적용된 클래스는 빈의 정의를 포함하는 메서드를 정의할 수 있습니다. 이 메서드는 @Bean 어노테이션과 함께 사용되어 빈의 생성과 구성을 정의합니다.

3. 다른 설정 클래스를 임포트할 수 있습니다: @Configuration 어노테이션이 적용된 클래스는 @Import 어노테이션을 사용하여 다른 설정 클래스를 임포트할 수 있습니다. 이를 통해 여러 설정 클래스를 조합하여 애플리케이션의 전체적인 구성을 정의할 수 있습니다.
